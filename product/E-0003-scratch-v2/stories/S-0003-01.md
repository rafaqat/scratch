Define the `.db.md` format for markdown-native databases.

## Acceptance Criteria

- [ ] Database definition file with YAML frontmatter: columns, types, views
- [ ] Column types: text, number, date, select, multi-select, checkbox, relation, url
- [ ] Each database = folder with `_schema.md` + row `.md` files
- [ ] Row files have frontmatter matching schema + optional markdown body
- [ ] Rust backend parses, validates, and CRUDs database rows
- [ ] Schema migrations: add/remove/rename columns updates existing rows

## Technical Notes

Generalize `src-tauri/src/stories.rs` kanban pattern into reusable database engine.

```
my-tasks/
  _schema.md       # columns, views, metadata
  row-001.md       # frontmatter fields + markdown body
  row-002.md
```
