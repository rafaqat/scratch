Plugin system that lets AI agents and users extend Scratch with new MCP tools.

## Acceptance Criteria

- [ ] `.scratch/plugins/` directory for plugin manifests (YAML)
- [ ] Plugin manifest defines: name, description, MCP tools, webhook handlers
- [ ] Custom MCP tools defined in plugins are auto-registered on server start
- [ ] Tool implementations can be: shell commands, HTTP calls, or note operations
- [ ] Plugin enable/disable in settings UI
- [ ] Plugin validation on load (schema check, permissions review)

## Technical Notes

Plugin manifest format:
```yaml
name: github-sync
version: 1.0.0
description: Sync GitHub issues to Scratch notes
tools:
  - name: github_sync_issues
    description: "Fetch open issues from a GitHub repo"
    params:
      repo: {type: string, required: true}
    action:
      type: shell
      command: "gh issue list --repo {{repo}} --json title,body,number"
      output: create_notes
webhooks:
  - path: github
    events: [issues.opened, issues.closed]
```
