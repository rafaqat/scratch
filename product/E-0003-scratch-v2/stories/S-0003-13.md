HTTP endpoint that accepts POST webhooks and creates/updates notes.

## Acceptance Criteria

- [ ] `POST /webhooks/{plugin-name}` endpoint on MCP server
- [ ] Configurable webhook handlers in `.scratch/plugins/`
- [ ] Handler defines: trigger conditions, note creation/update rules, field mapping
- [ ] Webhook payload accessible as variables in templates
- [ ] Secret-based authentication (per-webhook shared secret)
- [ ] Webhook activity log viewable in settings

## Technical Notes

Add routes to the existing Axum server in `src-tauri/src/mcp.rs`. Handler config is YAML:
```yaml
# .scratch/plugins/github-webhook.yaml
webhook:
  path: github
  secret: ${GITHUB_WEBHOOK_SECRET}
  on:
    issues.opened:
      action: create_note
      folder: github-issues
      template: |
        # {{payload.issue.title}}
        {{payload.issue.body}}
```
